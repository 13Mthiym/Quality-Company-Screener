# Stock Quality Screener

This project implements a machine learning model to classify S&P 500 stocks as "High Quality" or "Standard Quality" based on fundamental financial metrics. It uses a Random Forest classifier trained on historical data.

## Project Objective

To build a Random Forest classification model that categorizes S&P 500 stocks based on the following criteria for "High Quality":
- Return on Equity (ROE) > 15%
- Debt-to-Equity (D/E) Ratio < 0.5
- Price-to-Earnings (P/E) Ratio < 20 (and P/E > 0)

## Project Structure

The project is organized into several Python scripts:

-   `data_acquisition.py`: Fetches S&P 500 tickers, their GICS Sector, and Company Name, along with historical financial data (ROE, D/E) and current P/E from Yahoo Finance using the `yfinance` library.
-   `data_preprocessing.py`: Loads the raw financial data, cleans it (handles missing values), and engineers the "Quality" target variable based on the defined criteria.
-   `model_training.py`: Takes the preprocessed data, splits it into training and testing sets, trains a Random Forest classification model, and saves the trained model and the test datasets.
-   `model_evaluation.py`: Loads the trained model and test data, evaluates the model's performance (accuracy, confusion matrix, classification report), and analyzes feature importances.
-   `main.py`: Orchestrates the entire workflow by calling functions from the other modules in sequence. This is the main script to run the project.
-   `requirements.txt`: Lists all the Python dependencies required for the project.
-   `sp500_financial_data.csv`: (Generated by `data_acquisition.py` via `main.py`) Stores the raw financial data fetched, including Ticker, **Company**, Year, ROE, D/E, P/E, Report_Date, and Sector.
-   `processed_stock_data.csv`: (Generated by `data_preprocessing.py` via `main.py`) Stores the cleaned data with the engineered "Quality" feature, **Company**, and Sector information.
-   `random_forest_quality_model.joblib`: (Generated by `model_training.py` via `main.py`) The saved trained Scikit-learn Random Forest model.
-   `X_test.csv`, `y_test.csv`: (Generated by `model_training.py` via `main.py`) The test dataset features and labels, respectively.
-   `confusion_matrix.png`: (Generated by `model_evaluation.py` via `main.py`) A plot of the confusion matrix for the model's performance on the test set.
-   `feature_importances.png`: (Generated by `model_evaluation.py` via `main.py`) A plot of the feature importances from the model.

## Prerequisites

-   Python 3.x
-   `pip` for installing Python packages

## Setup and Installation

1.  **Clone the repository (if applicable) or ensure all project files are in the same directory.**

2.  **Create a virtual environment (recommended):**
    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate
    ```

3.  **Install the required Python packages:**
    Open your terminal or command prompt in the project's root directory and run:
    ```bash
    pip install -r requirements.txt
    ```

## How to Run

To run the entire workflow (data acquisition for a sample set, preprocessing, training, and evaluation):

1.  Ensure you have completed the setup steps above.
2.  Navigate to the project's root directory in your terminal.
3.  Run the `main.py` script:
    ```bash
    python main.py
    ```

This will:
-   Fetch financial data for a default sample of S&P 500 companies for the last 5 fiscal years (as defined in `main.py`).
-   Preprocess the data and create the 'Quality' labels.
-   Train a Random Forest model.
-   Evaluate the model and print metrics, including feature importances.
-   Generated files (CSVs, model, plots) will be saved in the project directory.

**Note on Data Acquisition:**
-   The `main.py` script by default runs with a small sample of tickers and 5 years of data for quicker execution.
-   To run for the full S&P 500 list or a custom set of tickers/years, you can modify the `run_full_workflow` call within the `if __name__ == '__main__':` block in `main.py`. Fetching data for the entire S&P 500 can be time-consuming.
-   The `yfinance` library relies on Yahoo Finance data, which is free but can sometimes have inconsistencies or missing data for certain tickers or historical periods.
